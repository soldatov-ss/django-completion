# 0.1.0 (2026-04-26)

Initial release.

## Features

- Tab completion for `manage.py` in bash and zsh
- Completes management command names, app labels, and option flags
- zsh shows command and option descriptions alongside completions
- Completion works for `manage.py`, `./manage.py`, `python manage.py`, `python3 manage.py`, and `python ./manage.py`
- Local JSON cache at `{project_root}/.django-completion-cache.json` — no network calls, no database access at tab-press time
- Auto-refresh: cache rebuilds in a background thread after each `manage.py` command (60-second cooldown)
- `DJANGO_COMPLETION_AUTO_REFRESH = False` setting to disable background auto-refresh

## Management commands

- `python manage.py autocomplete install` — installs shell hook and builds cache immediately
- `python manage.py autocomplete status` — shows cache age, command/app counts, and per-shell hook status
- `python manage.py autocomplete refresh` — force cache rebuild on demand
- `python manage.py autocomplete uninstall` — removes shell hook, managed script files, and install directory

## Limitations

- bash and zsh only
- requires `django_completion` in `INSTALLED_APPS`
- no migration-name completion (planned for v2)
- no `django-admin` support
- no native Windows/PowerShell support
